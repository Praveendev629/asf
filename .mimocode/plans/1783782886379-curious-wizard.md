# ASF Shopee - Full Enhancement Plan

## Overview
Transform the current single-auth grocery app into a **3-zone system**: Customer Store, Admin Dashboard (no auth — private), and Delivery Partner Portal (self-register). Replace Leaflet with Google Maps. Add delivery time prediction, user management, and navigation.

---

## 1. Admin Dashboard (No Login Required)

### Problem
Admin currently uses Google OAuth. User wants direct access — no login since it's a private link.

### Solution
- Remove all auth checks from admin page and admin API routes
- Admin page loads directly at `/admin`
- Admin API routes skip authentication entirely
- Keep the admin section hidden from customer navbar (only show when URL is /admin or via env-based check)
- Add "Delivery Partners" tab and "Users" tab to admin

## 3. Delivery Partner Portal

### New Model: `src/lib/models/DeliveryPartner.ts`
```
- name: string (required)
- phone: string (required)
- email: string (required, unique)
- password: string (hashed with bcryptjs)
- isAvailable: boolean (default true)
- currentLocation: { lat: number, lng: number }
- createdAt, updatedAt
```

### New Pages
- `src/app/delivery/layout.tsx` — Separate layout (no customer navbar, no cart, delivery-themed)
- `src/app/delivery/register/page.tsx` — Self-registration form (name, phone, email, password)
- `src/app/delivery/login/page.tsx` — Login form (email, password)
- `src/app/delivery/page.tsx` — Dashboard:
  - Shows available (unassigned) orders
  - Shows currently assigned/active orders to this partner
  - Each order shows: customer name, phone, delivery address, pinned map location
  - "Navigate" button → opens Google Maps directions from delivery partner to customer
  - "Accept Order" / "Mark Delivered" / "Update Status" buttons
  - Shows delivery partner's own current location on map
- `src/app/delivery/orders/[id]/page.tsx` — Single order detail with full map navigation

### How It Works
1. Delivery partner self-registers at `/delivery/register`
2. Logs in at `/delivery/login` (JWT stored in cookie/localStorage)
3. Sees available orders on dashboard
4. Clicks "Accept Order" → order's `deliveryPartner` field updates with partner's name + phone
5. Customer's order page auto-shows delivery partner info (already exists in Order model)
6. Partner clicks "Navigate" → opens Google Maps with directions to customer
7. Partner updates status (confirmed → packed → dispatched → out_for_delivery → delivered)
8. Partner's phone number auto-appears on the customer's order page and in admin

### New API Routes
- `src/app/api/delivery/auth/register/route.ts` — POST register
- `src/app/api/delivery/auth/login/route.ts` — POST login
- `src/app/api/delivery/orders/route.ts` — GET available + assigned orders
- `src/app/api/delivery/orders/[id]/route.ts` — PATCH accept order, update status
- `src/app/api/delivery/location/route.ts` — POST update current location
- `src/app/api/admin/delivery-partners/route.ts` — GET all delivery partners (for admin dashboard)

## 5. Delivery Time Prediction

### New Utility: `src/lib/distance.ts`
- Haversine formula for distance calculation
- ETA estimation: ~2 min per km average (configurable)
- Export functions: `haversineDistance(lat1, lng1, lat2, lng2)` and `estimateDeliveryTime(distanceKm)`

### Integration
- When delivery partner accepts order, calculate distance from partner to customer
- Store ETA in `deliveryPartner.eta` field (already exists in Order model)
- Display ETA on:
  - Customer order tracking page (`src/app/orders/[id]/page.tsx`) — prominent display
  - Admin orders list
  - Delivery partner dashboard
